Conteúdo detalhado

🟢 O que é

O sistema AgenteMktCX usa o Codex como uma "equipe de trabalho autônoma". 5 agentes especializados operam em paralelo, cada um com sua skill, pasta de output e regras de coordenação.

💡 Por que aprender

Uma campanha de marketing completa (pesquisa + estratégia + criativo + análise) pode ser produzida pelo Codex em minutos, com outputs organizados em pastas — sem ferramentas externas ou freelancers.

🔑 Os 5 agentes

AgentePapelOutput
Campaign ManagerOrquestradortask_manifest.json, relatório final
Research AgentPesquisador/output/research/
Content StrategyEstrategista/output/strategy/
Creative CopyRedator criativo/output/copy/
Data AnalysisAnalista de dados/output/data/

🟢 O que é

O sistema organiza tudo em arquivos de contexto compartilhados: system.md define as regras de orquestração, brand.md define a identidade da marca, e cada agente tem seu SKILL.md próprio.

🔑 Prompt 1 — Criar system.md

Crie system.md com este prompt no Codex:
Crie um arquivo system.md que defina:
- Funções de cada agente (Campaign Manager, Research,
  Content Strategy, Creative Copy, Data Analysis)
- Estrutura das SKILLs (SKILL.md por agente)
- Pastas /output por agente
- Regras de orquestração e handoff
- Formato do task_manifest.json para coordenação

🔑 Prompt 2 — Criar brand.md

Crie brand.md com:
- nome: [nome da marca]
- público: [perfil do cliente ideal]
- tom: [ex: profissional, jovem, técnico]
- posicionamento: [proposta de valor]
- temas: [tópicos de conteúdo]
- palavras-chave: [keywords SEO]
- canais: [LinkedIn, Instagram, email...]
- metas: [leads, brand awareness, vendas]

🟢 O que é

O Campaign Manager é o "diretor de campanha" — lê o brand.md, cria o task_manifest.json com as tarefas de cada agente e, ao final, consolida todos os outputs em um relatório executivo.

🔑 Prompt de ativação do Campaign Manager

Você é o Campaign Manager. Leia brand.md e crie
task_manifest.json com as tarefas para cada agente.
Depois, ao receber os outputs deles, gere um relatório
final de campanha consolidado em /output/campaign-report.md

🔑 Estrutura do task_manifest.json

{
  "campaign": "Q3 Lead Generation",
  "tasks": [
    {
      "agent": "research",
      "task": "Analise os 3 principais concorrentes",
      "output": "output/research/competitors.md",
      "status": "pending"
    },
    {
      "agent": "strategy",
      "task": "Crie plano de conteúdo para 30 dias",
      "output": "output/strategy/content-plan.md",
      "status": "pending",
      "depends_on": ["research"]
    }
  ]
}

🟢 Research Agent

O Research Agent analisa o mercado: busca dados de concorrentes, tendências do setor e oportunidades. Salva tudo organizado em /output/research/.

🔑 SKILL.md do Research Agent

Prompt 3 — criar agents/research/SKILL.md
Crie SKILL.md para o Research Agent que:
- Analisa os 3 principais concorrentes da marca
- Coleta dados do mercado (tamanho, crescimento)
- Identifica lacunas e oportunidades
- Salva relatório em /output/research/
- Formato: markdown estruturado com fontes

🟢 Content Strategy Agent

Usa os dados do Research para criar um plano de conteúdo estratégico: calendário de 30 dias, temas por canal, pilares de conteúdo alinhados com brand.md.

SKILL.md do Content Strategy
Crie SKILL.md para o Content Strategy Agent:
- Lê /output/research/ e brand.md
- Cria calendário de conteúdo (30 dias)
- Define pilares por canal (LinkedIn, IG, email)
- Salva em /output/strategy/content-plan.md

🟢 Creative Copy Agent

Transforma o plano de conteúdo em peças prontas para publicar: posts para cada canal, headlines para email, variações A/B para testar. Usa o tom definido em brand.md.

SKILL.md do Creative Copy Agent
Crie SKILL.md para o Creative Copy Agent:
- Lê brand.md e /output/strategy/content-plan.md
- Escreve posts completos para LinkedIn, Instagram
- Cria 3 variações de subject line para email
- Tom: conforme brand.md
- Salva tudo em /output/copy/

🟢 Data Analysis Agent

Analisa resultados de campanhas anteriores, identifica padrões de performance e sugere otimizações baseadas em dados. Entrega relatórios de KPIs em /output/data/.

SKILL.md do Data Analysis Agent
Crie SKILL.md para o Data Analysis Agent:
- Analisa dados de campanhas anteriores (CSV/JSON)
- Calcula CTR, conversão, CAC, ROAS
- Identifica top 3 conteúdos por engajamento
- Recomendações de otimização
- Salva relatório em /output/data/analysis.md

🟢 O que é

O task_manifest.json é o "quadro de tarefas" da equipe. O Campaign Manager o cria, os agentes atualizam seus status e o manifesto garante que o fluxo de trabalho seja respeitado (ex: strategy só roda após research).

🔑 Execução em paralelo vs sequencial

🔀 Em PARALELO (sem dependência)
  • • Research + Data Analysis juntos
  • • Rodam simultaneamente no Codex
  • • Usa subagentes do Codex
⬇️ Em SEQUÊNCIA (com dependência)
  • • Research → Content Strategy
  • • Strategy → Creative Copy
  • • Todos → Campaign Manager (final)

🔑 Comando de execução da campanha

codex "Leia task_manifest.json e execute todas as tarefas
pendentes. Para tarefas sem dependência, execute em paralelo.
Atualize o status de cada tarefa no manifesto e salve os
outputs nas pastas corretas. Ao finalizar tudo, gere
campaign-report.md com resumo executivo."

Resultado: Campanha completa em /output/ — pesquisa, estratégia, posts prontos, análise de dados e relatório executivo. Tudo organizado, tudo versionável no git.
